> Hi,

> ich wollte mal ein bisschen weiter in tel.py reinschnuppern und dabei

Ich habe keine Tabulatoren verwendet. Tabulatoren und somit die Einrückung 
sind ja Einstellungssache und so würde der Code von den Einstellungen 
abhängen. Das gefällt mir nicht... Außerdem gibt es einen Python Style 
Guide, den PEP 8 http://www.python.org/dev/peps/pep-0008/, und der sagt:

> For new projects, spaces-only are strongly recommended over tabs.  Most
> editors have features that make this easy to do.

Deswegen verwende ich vier Leerzeichen für jede Einrückungsebene.

> Da wir hier von Python Code reden kann und wird dies Probleme
> verursachen, da ja der Interpreter das korrekte Einrücken _erzwingt_!
> Zudem ist korrektes Einrücken sowieso besser leserlich ^^

ack, deswegen vier Leerzeichen. 

> Was machen wir da? Ist das eine Einstellung von vim? Du nutzt ja emacs.

Mmh, es scheint, dass vi automatisch immer Tabulatoren verwendet.
Das müsstest du auf vier Leerzeichen umstellen. Im Code von tel ist kein 
Tabulator drinnen, sondern nur Leerzeichen.

Zum Thema Vim und Python kannst du dir mal den Artikel im Python Wiki 
durchlesen, da wird auch erklärt, wie du vim konfigurierst:
http://wiki.python.de/Python-Programmieren_mit_Vim?highlight=%28vim%29

hope it helps
