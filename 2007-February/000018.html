Nachtrag:

Ich habe mir die Website mal heruntergeladen und auf meinem lokalen 
Webserver getestet. 
Dabei ist mir aufgefallen, dass Berlios für php offenbar alle Warnungen 
deaktiviert hat. Zumindest kamen bei mir Warnung, wenn man index.php ohne 
to aufruft. Daraufhin habe ich die Reihenfolge der if Abfragen in der Datei 
etwas verändert, damit diese Warnungen nicht mehr kommen.

Außerdem habe ich die Navigationsleiste am linken Rand so verändert, dass 
nun eine Liste <ul></ul> verwendet wird.
Damit die Listenzeichen verschwinden und außerdem die Links im Menü zur 
besseren Lesbarkeit einen größeren Abstand voneinander haben, gibts im 
Stylesheet eine neue Klasse "navigation", die sowohl für die Liste als auch 
für die Links im Inhaltsverzeichnis verwendet werden muss.
Die Liste wird in header.php erzeugt. Die Navigationslinks werden in der 
jeweiligen ihv.php nun so erzeugt:
<li><a class="navigation" href="blubb">test</a></li>
Ich denke, so sieht die Seite besser aus... Einen Screenshot im Firefox habe 
ich mal angehängt.

Außerdem hab ich noch einen Link zu Übersicht eingefügt, damit man wieder 
dahin zurückfindet. ;)

Leider kann ich das alles nicht hochladen... scp bringt mir ein "Permission 
denied" :( Offenbar ein Rechte-Problem...
Ich würde deswegen vorschlagen, dass wir die Website auch ins svn einfügen 
(als vierter Ordner website neben trunk, branches und tags). Dann kann 
jeder seine Änderungen dorthin commiten. Sobald die Website dann 
aktualisiert wurden, ziehst du dir die neueste Version und kopierst sie auf 
den Server. So vermeiden wir das Rechteproblem, ohne dass du jedes mal ein 
chmod nach dem Hochladen ausführen musst. Außerdem verhindern wir so, dass 
einer versehentlich die Änderungen des anderen überschreibt. Was hälst du 
davon?
