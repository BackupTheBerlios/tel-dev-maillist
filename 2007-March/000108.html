Hi @all interested

ich habe jetzt ein Installationsskript für tel geschrieben. Dieses Skript 
bietet Unterstützung für Internationalisierungsdateien (message Kataloge). 
tel.py spricht also zum ersten Mal Deutsch! Diese Skript installiert tel 
nach folgendem Muster:

Die Quelldateien (momentan nur tel.py) wandern nach $PREFIX/share/tel/
Um tel ohne Angabe eines Pfades ausführen zu können, wird ein symbolischer 
Link erzeugt, der von PREFIX/bin/tel nach $PREFIX/share/tel/tel.py zeigt. 
Außerdem werden die message Kataloge unter $PREFIX/share/locale 
installiert, und zwar nach folgendem Muster:
$PREFIX/share/locale/$SPRACHE/LC_MESSAGE/tel.mo

$PREFIX steht dabei für das Installationsprefix, welches standardmäßig /usr 
ist. Allerdings kann man es mittels der --prefix Option auch umstellen, 
z.B. auf /usr/local.

Außerdem erleichtert das Setupskript das Erstellen eines Releases, da das 
Packen in ein Archiv mit allen benötigen Dateien von einem komfortablen 
Befehl übernommen wird (python setup.py sdist).

Ich habe ein Preview auf den FTP Server [1] hochgeladen. Bitte probiert es 
aus, bevor wir ein neues Release erstellen. Ich habe es zwar schon etwas 
getestet, aber man weiß ja nie...

Die Installation ist ganz einfach. Das Archiv entpacken, in den Ordner 
wechseln und folgendes ausführen:
    
    python setup.py install

Jetzt wird tel unter /usr installiert. Wer sich sein Prefix selbst nehmen 
möchte, verwendet

    python setup.py install --prefix /usr/local

Beide Befehl benötigen Root Rechte. Wer meiner Programmierkunst nicht so 
traut ^^ und lieber erstmal langsam anfängt, kann das root verzeichnis 
umbiegen:

    python setup.py install --root ~/test

Dieses Kommando installiert alle Dateien unter ~/test. Die ausführbare Datei 
ist also unter ~/test/usr/bin/tel zu finden.

Have fun!
