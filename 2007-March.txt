Die Quelldateien (momentan nur tel.py) wandern nach $PREFIX/share/tel/
Um tel ohne Angabe eines Pfades ausf?hren zu k?nnen, wird ein symbolischer 
Link erzeugt, der von PREFIX/bin/tel nach $PREFIX/share/tel/tel.py zeigt. 
Au?erdem werden die message Kataloge unter $PREFIX/share/locale 
installiert, und zwar nach folgendem Muster:
$PREFIX/share/locale/$SPRACHE/LC_MESSAGE/tel.mo

$PREFIX steht dabei f?r das Installationsprefix, welches standardm??ig /usr 
ist. Allerdings kann man es mittels der --prefix Option auch umstellen, 
z.B. auf /usr/local.

Au?erdem erleichtert das Setupskript das Erstellen eines Releases, da das 
Packen in ein Archiv mit allen ben?tigen Dateien von einem komfortablen 
Befehl ?bernommen wird (python setup.py sdist).

wechseln und folgendes ausf?hren:
    
    python setup.py install

Jetzt wird tel unter /usr installiert. Wer sich sein Prefix selbst nehmen 
m?chte, verwendet

    python setup.py install --prefix /usr/local
